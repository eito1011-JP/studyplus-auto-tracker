# studyplus-auto-tracker

開発時間を [ActivityWatch](https://activitywatch.net/) で自動計測し、確定した作業ブロックを [StudyPlus](https://www.studyplus.jp/) に自動記録する macOS 向けの個人ツールです。

**「記録の開始ボタンを押し忘れる」問題を、ボタン操作そのものを無くして解決します。**

---

## なぜ作ったか

StudyPlus に開発時間を記録したいのに、毎回「開始ボタン」を押すのを忘れる。タブに埋もれて押さなくなる。原因は「記憶に頼っていること」と「記録のトリガーが作業の動線から外れていること」でした。

そこで、**普段どおり開発しているだけで稼働が自動で計測され、まとまった作業ブロックが勝手に記録される**ようにしました。もうボタンは押しません。

## 仕組み

```
ActivityWatch（稼働を計測・無操作で自動的に区切る）
        │  ローカル API
        ▼
ポーラー（launchd で数分おきに実行）
   ・対象アプリ（エディタ / ターミナル / 開発ブラウザ）の稼働を抽出
   ・短い休憩はつなぎ、15分以上空いたら別ブロックに区切る
   ・末尾が15分以上途切れた「確定ブロック」だけを投稿対象にする
   ・記録済み台帳と照合 → 未記録だけを抽出
        │
        ▼
StudyPlus に記録を作成（作業の実際の開始時刻・実稼働時間で登録）
        │
        ▼
通知（失敗は即通知 + 1日1回サマリー）
```

二重記録は「自前の台帳」と「ブロックごとに決まる冪等トークン」の二段構えで防ぎます。

## 必要なもの

- **macOS**（通知・常駐・トークン保管に macOS の仕組みを使います）
- **Node.js 20+**
- **[ActivityWatch](https://activitywatch.net/)**（稼働計測エンジン）
- **StudyPlus アカウント**

## セットアップ

### 1. ActivityWatch を入れて起動する

```bash
brew install --cask activitywatch
open -a ActivityWatch
```

起動後、`aw-watcher-window` がアプリ名とウィンドウタイトルを取得できるよう、
**システム設定 → プライバシーとセキュリティ → アクセシビリティ / 画面収録** で ActivityWatch を許可してください（タイトルが取れないと開発ブラウザの判定ができません）。

ログイン時に自動起動させたい場合は、メニューバーの ActivityWatch アイコンから **Open on login** を有効にしておくと安心です。

### 2. このツールを入れる

```bash
git clone https://github.com/eito1011-JP/studyplus-auto-tracker.git
cd studyplus-auto-tracker
npm install
```

### 3. StudyPlus 側に教材を用意する

記録はすべて1つの教材に集約します。StudyPlus で **「アプリ開発」という名前の教材**を作っておいてください（名前を変えたい場合は [設定](#設定) を参照）。

### 4. アクセストークンを登録する

ブラウザの StudyPlus からトークンを取り出して登録します（初回のみ）。

1. ブラウザで StudyPlus にログイン
2. デベロッパーツール → **Network** タブを開く
3. `api.studyplus.jp` 宛のリクエストを1つ選び、**Request Headers** の
   `authorization: OAuth <ここの値>` をコピー
4. 登録：

```bash
npm run seed
# プロンプトに貼り付けて Enter（疎通確認に成功すると保管されます）
```

トークンは **macOS のキーチェーン**に保管され、リポジトリには一切書き込まれません。
後でトークンが失効すると通知が出るので、その時はもう一度 `npm run seed` を実行します。

### 5. 動作確認して常駐させる

```bash
npm run poll            # 1回だけ手動実行して記録されるか確認
npm run launchd:install # 問題なければ常駐（5分間隔）を有効化
```

常駐の停止は `npm run launchd:uninstall`。ログは `.state/poll.log` に出ます。

## 設定

計測対象や閾値は [`src/config.ts`](./src/config.ts) で変更できます。

| 設定 | 意味 | 既定 |
|---|---|---|
| `alwaysCountApps` | アプリ名だけで対象にする（エディタ・ターミナル） | `Code`, `Cursor`, `Ghostty` |
| `titleGatedApps` | タイトルが開発関連のときだけ対象にする（ブラウザ） | `Dia` |
| `devTitlePatterns` | 「開発関連タイトル」とみなす正規表現 | GitHub / localhost / 技術ドキュメント等 |
| `idleGapMs` | この時間未満の空きは同じブロックに連結 | 15分 |
| `finalizeSilenceMs` | 末尾がこの時間途切れたらブロック確定 | 15分 |
| `materialName` | 記録を紐づける教材名 | `アプリ開発` |

> アプリ名は ActivityWatch が報告する名前です（例：VS Code は `Code`）。
> 自分の環境での名前は ActivityWatch の Web UI（http://localhost:5600）で確認できます。

## 記録のルール

- **無操作の扱い**：ActivityWatch はキーボード/マウスの最後の操作から **約3分（既定）** までは「作業中」とみなします。数十秒〜数分の「画面を読む・考える」時間は記録に含まれ、3分を超える完全な無操作だけが休憩として外れます。
- **ブロック化**：対象アプリの作業を時系列にまとめ、空きが15分未満ならつなぎ、15分以上で区切ります。
- **確定**：あるブロックの末尾から15分以上途切れて初めて「確定」とし、記録対象にします（作業中のブロックは記録されません）。
- **記録内容**：作業の**実際の開始時刻**と**実稼働時間**で登録します。

## データの扱い

- アクセストークンは **macOS キーチェーン**にのみ保管します。
- 記録済み台帳や状態は `.state/` 配下のローカルファイルです（git 管理外）。
- 通信先はローカルの ActivityWatch と StudyPlus だけで、ほかの外部サーバーには何も送りません。

## できること / まだやらないこと

**できること**：稼働の自動計測・ブロック化・StudyPlus への自動記録・失敗通知・常駐実行。

**まだやらないこと（割り切り）**：
- プロジェクトごとに教材を分ける（今は1教材に集約）
- 複数マシンの統合（ActivityWatch はマシン単位。メイン1台前提）
